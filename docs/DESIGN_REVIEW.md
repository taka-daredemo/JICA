# デザイン・実装確認ガイド

## 🎨 現在のデザインシステム

### カラーパレット
- **Primary (Blue)**: `bg-blue-600` - 主要なアクションボタン、アクティブ状態
- **Secondary (Slate)**: `bg-slate-800` - セカンダリボタン
- **Success (Green)**: `bg-green-100 text-green-700` - 完了ステータス
- **Warning (Orange/Yellow)**: `bg-orange-100 text-orange-700` / `bg-yellow-100 text-yellow-700` - 警告、高優先度
- **Danger (Red)**: `bg-red-100 text-red-700` / `bg-red-600` - エラー、期限切れ、緊急
- **Info (Blue)**: `bg-blue-100 text-blue-700` - 情報表示

### レイアウト
- **コンテナ**: `max-w-7xl` (1280px)
- **スペーシング**: Tailwind標準（8pxベース）
- **レスポンシブ**: Mobile-first設計
- **背景**: `bg-slate-50` / `bg-gray-50`

### タイポグラフィ
- **見出し**: `text-3xl font-bold` (H1), `text-xl font-semibold` (H2)
- **本文**: `text-sm` (14px), `text-base` (16px)
- **カラー**: `text-gray-900` (見出し), `text-gray-600` (本文), `text-gray-500` (補助テキスト)

### コンポーネントスタイル
- **カード**: 白背景、ボーダー、角丸、影なし（シンプル）
- **ボタン**: 角丸 `rounded-md`, 複数バリアント
- **バッジ**: 角丸 `rounded-full`, 小さなパディング

## 📄 実装済みページの確認方法

### 1. ダッシュボード (`/ja/dashboard` または `/en/dashboard`)

**特徴**:
- 独自のヘッダー（MainLayoutを使用していない）
- 統計カード（4列グリッド）
- アラート表示エリア
- 最近のタスクと今後のイベント（2列グリッド）

**確認ポイント**:
- ✅ 実データ連携済み（APIからデータ取得）
- ✅ レスポンシブ対応（モバイル/タブレット/デスクトップ）
- ⚠️ MainLayoutを使用していない（統一性の問題）

**スクリーンショット確認項目**:
- 統計カードが正しく表示されているか
- アラートが適切に表示されているか
- 最近のタスクが表示されているか
- 言語切り替えが動作するか

## 🚀 開発サーバーでの確認方法

1. **開発サーバー起動**:
   ```bash
   npm run dev
   ```

2. **ブラウザでアクセス**:
   - `http://localhost:3000/ja/dashboard`
   - `http://localhost:3000/ja/tasks`
   - `http://localhost:3000/ja/budget`
   - など

3. **確認すべき項目**:
   - ページが正しく表示されるか
   - サイドバーのナビゲーションが動作するか
   - 言語切り替えが動作するか
   - 実データが表示されるか（ダッシュボード、タスク）

## 📝 改善提案

### デザイン統一性
1. **ダッシュボードページをMainLayoutに統一**
   - 現在、独自のヘッダーを使用しているため、統一感がない

2. **カラーシステムの明文化**
   - Tailwind設定ファイルにカスタムカラーを定義
   - デザインシステムドキュメントを作成

3. **コンポーネントライブラリの拡充**
   - バッジ、ステータス表示、アバターなどの再利用可能コンポーネント

### UI/UX改善
1. **ローディング状態の統一**
   - スケルトンローダーまたはスピナーの統一

2. **エラーハンドリングの表示**
   - エラーメッセージの統一デザイン

3. **空状態の表示**
   - データがない場合のメッセージとアクション
